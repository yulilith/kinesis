# 🦞 Claw的AI姿态监控系统

一个创新的分布式AI基础设施项目，通过ESP32"代理人"实现智能姿态监控和触觉反馈。

## 🎯 项目创新点

### Infrastructure创新
- **设备无关AI部署**：任何具备基础计算能力的设备都能成为AI的"手脚"
- **分布式智能架构**：边缘设备负责数据采集，中央AI负责复杂决策
- **实时反馈循环**：从感知到执行的完整闭环系统

### 技术特色
- **轻量级代理模式**：ESP32上运行的"AI片段"与树莓派大脑通信
- **多模态反馈**：振动、EMS刺激、LED视觉提示
- **智能姿态分析**：基于IMU数据的实时姿态评估

## 🔧 硬件需求

### 必需组件
- ESP32开发板 x1
- MPU6050 IMU传感器 x1
- 树莓派（或任何运行Linux的设备）x1
- 振动马达 x1
- LED x1

### 可选组件
- EMS刺激器（需要专业医疗级设备）
- 按钮开关 x1

## 📦 安装步骤

### 1. 安装系统依赖
```bash
chmod +x install_dependencies.sh
./install_dependencies.sh
```

### 2. Arduino IDE库安装
在Arduino IDE的库管理器中安装：
- `WebSockets` by Markus Sattler
- `ArduinoJson` by Benoit Blanchon
- `MPU6050` by Electronic Cats

### 3. 硬件连接

#### ESP32引脚连接：
```
MPU6050:
- VCC → 3.3V
- GND → GND  
- SDA → GPIO 21
- SCL → GPIO 22

输出设备:
- LED → GPIO 2 (内置LED)
- 振动马达 → GPIO 4
- EMS设备 → GPIO 5 (⚠️ 需要专业电路)
- 按钮 → GPIO 0 (内置BOOT按钮)
```

### 4. 配置WiFi
编辑 `esp32_agent.ino`，修改WiFi设置：
```cpp
const char* ssid = "你的WiFi名称";
const char* password = "你的WiFi密码";
const char* ai_server_ip = "树莓派IP地址";
```

### 5. 上传代码到ESP32
1. 连接ESP32到电脑
2. 在Arduino IDE中选择正确的板子和端口
3. 上传 `esp32_agent.ino`

### 6. 启动AI服务器
在树莓派上运行：
```bash
python3 ai_posture_server.py
```

## 🧠 AI分析功能

### 姿态检测
- **驼背检测**：基于俯仰角度判断
- **侧倾检测**：基于翻滚角度分析
- **久坐监控**：时间累计与活动检测
- **运动量评估**：陀螺仪数据分析

### 反馈策略
- **轻度提醒**：LED闪烁
- **中度警告**：振动反馈
- **严重警告**：EMS刺激（需要专业设备）
- **组合提醒**：多模态反馈

### 个性化学习
- 记录用户姿态历史
- 适应个人习惯模式
- 动态调整提醒阈值

## 📊 使用演示

### 启动系统
1. 确保ESP32和树莓派在同一WiFi网络
2. 先启动AI服务器：`python3 ai_posture_server.py`
3. 重启ESP32，查看连接日志

### 监控日志
AI服务器会显示：
```
🦞 - INFO - 新的ESP32代理连接: ('192.168.1.50', 54321)
🦞 - INFO - 姿态分析完成 - 评分: 85, 警告数: 1
🦞 - WARNING - 高严重性姿态警告: ['检测到驼背！前倾角度: 35.2°']
```

### 体验反馈
- 故意驼背 → 感受振动提醒
- 侧身倾斜 → 观察LED警告
- 长时间不动 → 接收活动建议

## 🔬 技术深入

### 分布式AI架构
这个系统展示了一种新的AI部署模式：
- **边缘智能**：ESP32处理实时数据采集和基础响应
- **云端大脑**：树莓派进行复杂分析和决策
- **动态负载**：根据任务复杂度分配计算资源

### 扩展可能性
- 添加更多ESP32节点实现多点监控
- 集成语音交互功能
- 连接智能家居设备
- 开发移动端监控App

### Infrastructure意义
这不是一个简单的健康设备，而是一个可扩展的"AI感知网络"框架：
- 任何设备都能快速获得AI能力
- 统一的通信协议和数据格式
- 模块化的功能组件

## ⚠️ 安全提醒

### EMS设备安全
- 仅使用经过认证的医疗级EMS设备
- 遵循设备说明书的强度和时间限制
- 有心脏疾病等健康问题的用户请勿使用
- 建议先从振动反馈开始测试

### 数据隐私
- 所有数据在本地网络内处理
- 不会上传到外部服务器
- 用户完全控制数据存储和删除
